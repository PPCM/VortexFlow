# Changelog - 2026-05-06

## Summary
3D rendering polish (auto-zoom, particle visibility, text occlusion fixes),
move the simulation entirely to the frontend, unify the toolbar and panel
simulation controls, and turn "Émission particules" into a one-shot trace
button. Documentation updated. Closing pass on Dependabot security alerts:
backend down to 0 vulnerabilities. Then full CRA → Vite migration on the
frontend, which dropped the audit count to 0 there too and cut dev server
boot from ~10s to ~200ms.

## Changes

- [frontend/src/components/graphs/GraphRenderer3D.tsx] Auto-zoom on open
  (snap-fit then animate to half-distance over 1s); opaque + DoubleSide
  materials on 3D node geometries to fix rotation flicker; SpriteText
  labels with `depthTest:false` + `renderOrder:10` so they stay on top
  of particles; runtime patch (200ms interval) forcing
  `transparent:false, opacity:1` on particle SphereGeometry materials;
  fixed-color particles (`#ffd54f`); `linkDirectionalParticles` returns
  0 unless `simulationRunning && showParticles`; `emitParticles` state
  removed (was redundant with `simulationRunning`); new
  `isSimulationRunning` + `onToggleSimulation` props; `setCurrentGraphData`
  now called in the live initializeGraph; per-frame stats fallback for
  graphs without VortexFlow attributes (scene particle count + average
  traversal latency from particleSpeed + topological in-degree as
  bottleneck count); new `handleEmitTrace` callback that fires one-shot
  cascading particles from emitter nodes, with cycle-protection visited
  set and depth cap 15; "Émission particules" panel button is now a
  one-shot trigger; "Start/Pause Simulation" panel button delegates to
  `onToggleSimulation` when provided.
- [frontend/src/components/graphs/GraphViewer.tsx] Pass
  `isSimulationRunning` and `onToggleSimulation` (new
  `handleToggleSimulation`) to GraphRenderer3D so toolbar and panel
  controls stay in sync.
- [frontend/src/context/GraphContext.tsx] Simulation actions made local:
  `startSimulation/stop/pause` no longer call `apiService` or
  `webSocketService`, they only flip the local `simulationState` flag.
- [frontend/src/context/GraphContext.test.tsx] Tests updated to assert
  no API/WebSocket calls and that the local flag is toggled correctly.
- [frontend/src/types/index.ts] `SimulationState.sessionId` field
  added (unused in the new flow but preserved for SimulationContext).
- [README.md] Added "Comportements 3D / simulation" subsection
  describing auto-zoom, conditional particles, real-time stats,
  toolbar/panel sync, and one-shot trace button. Annotated the
  WebSocket events list to note the simulation handler is dormant.
  Updated the Frontend stack to Vite + Vitest.
- [CLAUDE.md] Updated "WebSocket / simulation" to clarify simulation
  is browser-side; added a "GraphRenderer3D behaviors (load-bearing)"
  subsection. Replaced the "Frontend" command block with Vite/Vitest
  commands; updated env variable convention to `VITE_*`; documented
  the form-submit-in-jsdom workaround for tests.
- [backend/package.json] Bumped `bcrypt` 5.1.0 → 6.0.0 and
  `nodemailer` 6.9.3 → 8.0.7 to clear high-severity advisories;
  `npm audit` reports 0 vulnerabilities.
- [frontend/package.json] Removed `react-scripts`, `@types/jest`, and
  the CRA-specific eslint config / browserslist. Added
  `vite`/`@vitejs/plugin-react`/`vitest`/`@vitest/coverage-v8`/`jsdom`,
  bumped `@types/node` → 22, `typescript` → 5, kept
  `@testing-library/user-event` at v13 (v14's async API would require
  rewriting all tests). Switched scripts to `vite`/`vite build`/
  `vitest run`. Pruned all overrides except `nth-check`.
- [frontend/vite.config.ts] (new) Vite + Vitest config: dev server on
  :3000, build into build/, jsdom test environment with globals,
  `monaco-editor` aliased to a stub so tests resolve the import
  before `vi.mock` substitutes the real fake.
- [frontend/index.html] Moved from `public/index.html` to project root
  with a `<script type="module" src="/src/main.tsx">` entry; removed
  CRA's `%PUBLIC_URL%` placeholders.
- [frontend/src/main.tsx] Renamed from `index.tsx` (Vite entry).
- [frontend/src/vite-env.d.ts] (new) Declares `ImportMetaEnv` for
  `VITE_API_URL` / `VITE_WS_URL`.
- [frontend/src/test-stubs/empty.ts] (new) Stub module for editor
  packages aliased by Vitest.
- [frontend/src/services/api.ts, websocket.ts] `process.env.REACT_APP_*`
  → `import.meta.env.VITE_*`.
- [frontend/src/services/errorHandler.ts,
   frontend/src/components/common/ErrorBoundary.tsx]
  `process.env.NODE_ENV` → `import.meta.env.MODE` (avoids needing
  `@types/node` in browser code).
- [frontend/.env] Renamed `REACT_APP_API_URL/WS_URL` → `VITE_API_URL/WS_URL`.
- [frontend/tsconfig.json] target ES5 → ES2020, moduleResolution
  node → bundler, added `vite/client` + `vitest/globals` +
  `@testing-library/jest-dom` to `types`.
- [frontend/src/context/AuthContext.tsx] Replaced
  `NodeJS.Timeout` with `ReturnType<typeof setInterval>` (browser-safe).
- [frontend/src/**/*.test.ts(x)] Bulk migration `jest.*` → `vi.*`
  (fn, mock, spyOn, useFakeTimers, advanceTimersByTime,
  restoreAllMocks, etc.); `jest.requireActual` → `vi.importActual`;
  `jest.Mock` type → `Mock` from `vitest`; dropped CRA-only
  `{ virtual: true }` 3rd arg on `vi.mock`. `global.fetch` →
  `globalThis.fetch`. The auth pages' "rejects malformed email"
  tests use `fireEvent.submit(form)` instead of clicking the submit
  button, since jsdom + vitest does not fire the form's native
  submit event from a button click.
- [frontend/src/components/graphs/DOTCodeMirrorEditor.test.tsx]
  Replaced `LanguageSupport: vi.fn(() => 'support-stub')` with a real
  class — vi.fn with an arrow body cannot be used as a constructor.
- [frontend/public/index.html] Removed (replaced by /index.html at
  project root).
