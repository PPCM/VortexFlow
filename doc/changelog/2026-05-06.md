# Changelog - 2026-05-06

## Summary
3D rendering polish (auto-zoom, particle visibility, text occlusion fixes),
move the simulation entirely to the frontend, unify the toolbar and panel
simulation controls, and turn "Émission particules" into a one-shot trace
button. Documentation updated. Closing pass on Dependabot security alerts:
backend down to 0 vulnerabilities, frontend critical/runtime issues fixed
(only CRA dev-toolchain false positives remain).

## Changes

- [frontend/src/components/graphs/GraphRenderer3D.tsx] Auto-zoom on open
  (snap-fit then animate to half-distance over 1s); opaque + DoubleSide
  materials on 3D node geometries to fix rotation flicker; SpriteText
  labels with `depthTest:false` + `renderOrder:10` so they stay on top
  of particles; runtime patch (200ms interval) forcing
  `transparent:false, opacity:1` on particle SphereGeometry materials;
  fixed-color particles (`#ffd54f`); `linkDirectionalParticles` returns
  0 unless `simulationRunning && showParticles`; `emitParticles` state
  removed (was redundant with `simulationRunning`); new
  `isSimulationRunning` + `onToggleSimulation` props; `setCurrentGraphData`
  now called in the live initializeGraph; per-frame stats fallback for
  graphs without VortexFlow attributes (scene particle count + average
  traversal latency from particleSpeed + topological in-degree as
  bottleneck count); new `handleEmitTrace` callback that fires one-shot
  cascading particles from emitter nodes, with cycle-protection visited
  set and depth cap 15; "Émission particules" panel button is now a
  one-shot trigger; "Start/Pause Simulation" panel button delegates to
  `onToggleSimulation` when provided.
- [frontend/src/components/graphs/GraphViewer.tsx] Pass
  `isSimulationRunning` and `onToggleSimulation` (new
  `handleToggleSimulation`) to GraphRenderer3D so toolbar and panel
  controls stay in sync.
- [frontend/src/context/GraphContext.tsx] Simulation actions made local:
  `startSimulation/stop/pause` no longer call `apiService` or
  `webSocketService`, they only flip the local `simulationState` flag.
- [frontend/src/context/GraphContext.test.tsx] Tests updated to assert
  no API/WebSocket calls and that the local flag is toggled correctly.
- [frontend/src/types/index.ts] `SimulationState.sessionId` field
  added (unused in the new flow but preserved for SimulationContext).
- [README.md] Added "Comportements 3D / simulation" subsection
  describing auto-zoom, conditional particles, real-time stats,
  toolbar/panel sync, and one-shot trace button. Annotated the
  WebSocket events list to note the simulation handler is dormant.
- [CLAUDE.md] Updated "WebSocket / simulation" to clarify simulation
  is browser-side; added a "GraphRenderer3D behaviors (load-bearing)"
  subsection documenting the auto-zoom trick, particle-material patch,
  particle gating, toolbar/panel sync rule, one-shot trace semantics,
  stats fallback, and the dual-initializeGraph caveat.
- [backend/package.json] Bumped `bcrypt` 5.1.0 → 6.0.0 (used API
  `hash`/`compare` is unchanged) and `nodemailer` 6.9.3 → 8.0.7
  (API used `createTransport`/`sendMail`/`verify` is unchanged).
  `npm audit` reports 0 vulnerabilities (was 30, including 1 critical).
- [frontend/package.json] Extended `overrides` to force patched
  versions of transitive deps flagged by Dependabot:
  `serialize-javascript`, `underscore`, `form-data`, `node-forge`,
  `postcss`, `webpack`, `webpack-dev-server`, `path-to-regexp`, `qs`,
  `brace-expansion`, `socket.io-parser`. Critical CVE (form-data
  CVE-2025-7783) eliminated. Remaining alerts are CRA dev-toolchain
  noise that npm audit reports despite overrides being applied
  correctly (verified via `npm ls`).
