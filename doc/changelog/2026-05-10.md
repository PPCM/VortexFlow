# Changelog - 2026-05-10

## Summary
Initialized ruflo (claude-flow) inside the project to wire up Claude Code
hooks. The `.claude/helpers/hook-handler.cjs` referenced by the global
`~/.claude/settings.json` was missing locally, which produced
`PreToolUse:Bash hook error / cjs/loader:1404 (MODULE_NOT_FOUND)` lines on
every tool call. `ruflo init` deposited the helpers; the hooks now run
cleanly. Also refreshed `CLAUDE.md` (off-repo) to match the recent ESLint /
bundle-split / CI changes that landed last commits.

## Code fixes (chantier 3.7 — hardcoded URL)

- [frontend/src/components/graphs/GraphRenderer3D.tsx] Replaced the
  hardcoded `http://192.168.5.30:5000/api/public/parse-dot` URL with
  the same env-var pattern that `services/api.ts` already uses:
  `import.meta.env.VITE_API_URL || 'http://localhost:5000/api'`. The
  app is now portable across dev / prod / LAN deployments without
  per-machine code edits. Surgical change — kept the surrounding
  fetch + fallback logic untouched. Removed the `console.log` that
  echoed the URL (it was helpful for debugging the hardcoded host,
  no longer relevant). The renderer's own test suite (7 tests) still
  passes since it mocks `globalThis.fetch`, not the URL.

## Notes
- `ruflo init` is per-project. Each new project needs it once to silence
  the hook-handler error; daemon / memory / swarm init are optional and
  also per-project.
- Setup chosen here: manual mode (no auto-init wrapper around `cd`, no
  systemd-user supervisor). The supervisor would be mono-project (the
  unit's `WorkingDirectory` is hard-coded), which doesn't fit a
  multi-project workflow.
- The 5 orphaned validation reports under `backend/doc/` (FINAL_VALIDATION,
  TEST_REPORT, REDIS_INTEGRATION, TOOLKIT_SUMMARY, DOCUMENTATION_SUMMARY)
  were *unlinked* from the index but **not deleted** — keep them for
  history or remove them in a separate cleanup pass.
