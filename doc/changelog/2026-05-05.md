# Changelog - 2026-05-05

## Summary
Extended backend route test coverage to graph versions, shares, and duplication.

## Changes

- [backend/tests/integration/routes/graphs-extended.test.js] New: 23 tests for the rest of `/api/graphs` — PUT /:id (404, DOT validation rejection, validation errors, version bumping rules: dotCode/simulationConfig/visualSettings → bump+snapshot, title-only → no bump, explicit `createVersion=true`), GET /:id/versions, POST /:id/versions/:versionId/restore (404 on graph and on version, restore() invocation), POST /:id/share (validation, bad email, invalid permission, target user lookup, self-share guard, ALREADY_SHARED conflict, 201 with payload merge), DELETE /:id/shares/:shareId (404, revoke()), POST /:id/duplicate (title length validation, default "(Copy)" suffix, custom title).

- [backend/tests/unit/models/User.test.js] New: 11 tests — getFullName (both names / fallback to email / partial names), toJSON (strips password_hash), validatePassword (bcrypt hash round-trip), updateLastLogin (mutates + saves), class helpers (findByEmail lowercases, findActive, countByRole).
- [backend/tests/unit/models/Graph.test.js] New: 12 tests — canBeAccessedBy (public vs private with anonymous/owner/admin/other), canBeEditedBy (owner/admin only), incrementViewCount (atomic field save), updateLastSimulation, class helpers (findPublic, findByUser, findTemplates with/without category).
- [backend/tests/unit/models/GraphShare.test.js] New: 17 tests — isExpired (null / future / past), isValid (active vs revoked vs expired), incrementAccessCount, revoke, extend (24h default + custom), class helpers (findByGraph/User/Token, createShare with and without expiresIn computes correct expires_at, cleanupExpired).
- [backend/tests/unit/models/GraphVersion.test.js] New: 6 tests — class helpers (findByGraph/findLatestByGraph/findMajorVersions), createFromGraph snapshot logic, restore() updates target graph and creates a marker version, restore throws on missing graph.
- [backend/tests/unit/models/SimulationSession.test.js] New: 17 tests — lifecycle (start/pause/resume/complete/fail with duration computation, fail with Error and string), addFrame (timestamp + 1000-frame rolling cap), updateMetrics (shallow merge), addEvent, export (json/csv flatten frame metrics), class helpers (findByGraph/User/Running, createSession seeds session_data, cleanupOldSessions).

## Notes
Models are tested without Postgres by using `Model.build()` (no save) and stubbing `save()`/`create()`/`findAll()`/etc. via `jest.spyOn`. Backend test suite total now stands at 360 (173 unit + 187 integration). All routes mounted under `/api` in `server.js` are under test, plus the public-method surface of every Sequelize model.
