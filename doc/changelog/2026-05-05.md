# Changelog - 2026-05-05

## Summary
Extended backend route test coverage to graph versions, shares, and duplication.

## Changes

- [backend/tests/integration/routes/graphs-extended.test.js] New: 23 tests for the rest of `/api/graphs` — PUT /:id (404, DOT validation rejection, validation errors, version bumping rules: dotCode/simulationConfig/visualSettings → bump+snapshot, title-only → no bump, explicit `createVersion=true`), GET /:id/versions, POST /:id/versions/:versionId/restore (404 on graph and on version, restore() invocation), POST /:id/share (validation, bad email, invalid permission, target user lookup, self-share guard, ALREADY_SHARED conflict, 201 with payload merge), DELETE /:id/shares/:shareId (404, revoke()), POST /:id/duplicate (title length validation, default "(Copy)" suffix, custom title).

- [backend/tests/unit/models/User.test.js] New: 11 tests — getFullName (both names / fallback to email / partial names), toJSON (strips password_hash), validatePassword (bcrypt hash round-trip), updateLastLogin (mutates + saves), class helpers (findByEmail lowercases, findActive, countByRole).
- [backend/tests/unit/models/Graph.test.js] New: 12 tests — canBeAccessedBy (public vs private with anonymous/owner/admin/other), canBeEditedBy (owner/admin only), incrementViewCount (atomic field save), updateLastSimulation, class helpers (findPublic, findByUser, findTemplates with/without category).
- [backend/tests/unit/models/GraphShare.test.js] New: 17 tests — isExpired (null / future / past), isValid (active vs revoked vs expired), incrementAccessCount, revoke, extend (24h default + custom), class helpers (findByGraph/User/Token, createShare with and without expiresIn computes correct expires_at, cleanupExpired).
- [backend/tests/unit/models/GraphVersion.test.js] New: 6 tests — class helpers (findByGraph/findLatestByGraph/findMajorVersions), createFromGraph snapshot logic, restore() updates target graph and creates a marker version, restore throws on missing graph.
- [backend/tests/unit/models/SimulationSession.test.js] New: 17 tests — lifecycle (start/pause/resume/complete/fail with duration computation, fail with Error and string), addFrame (timestamp + 1000-frame rolling cap), updateMetrics (shallow merge), addEvent, export (json/csv flatten frame metrics), class helpers (findByGraph/User/Running, createSession seeds session_data, cleanupOldSessions).

- [backend/src/services/emailService.js] **Bug fix surfaced by tests**: `nodemailer.createTransporter(...)` was a typo — the real method is `createTransport` (singular). The catch block swallowed the resulting `TypeError`, so in production the email service silently failed: `isConfigured` flipped to false, and every send method returned `false` with a "service not configured" log. Fixed.
- [backend/tests/unit/services/emailService.test.js] New: 20 tests covering `setupTransporter` (missing env, full config, verify failure flips isConfigured), all four send methods (configured success/failure paths and not-configured fallthrough), `testConfiguration`, and template generators (welcome with role + dashboard link, fallback to "there", reset URL embed, share permission text, completion stats with `Math.round`).
- [backend/tests/unit/websocket/simulationHandler.test.js] New: 12 tests for the WebSocket coordinator — constructor wires `connection` event and registers the 8 lifecycle events on each socket, `handleJoinSimulation` (404 path, room join + state emit, current-state replay when active), `handleLeaveSimulation`, `handleDisconnection` (auto-pause when room empty, no-pause when others remain, no-op without sessionId), `cleanup()` (cancels every active session, resilient to per-session DB failures, no-op on empty map). SimulationEngine internals (DOT parsing, physics) are intentionally out of scope.
- [frontend/src/services/api.test.ts] New: 18 tests for the ApiService — auth (login/register response shape adaptation, getCurrentUser auth flag handling), graphs CRUD URL building, DOT validation endpoint routing, blob/multipart import-export options, user list pagination params. axios is mocked at module level with the singleton client recovered via `axios.create.mock.results[0].value`.
- [frontend/src/services/websocket.test.ts] New: 17 tests for WebSocketService — connect (factory creates a socket with the right URL and options), all emit-guarded methods (join/leave room, start/stop/pause simulation, updateSimulationConfig, updateGraph, sendCursorPosition, sendChatMessage, generic emit) verify "no emit when disconnected", utilities (setCallbacks merge, removeCallbacks, isSocketConnected, getSocketId, on/off forwarding).

## Notes
Models are tested without Postgres by using `Model.build()` (no save) and stubbing `save()`/`create()`/`findAll()`/etc. via `jest.spyOn`. Backend test suite total now stands at **392** (224 unit + 187 integration), frontend at **69**. The full surface of every backend module under `src/` (routes, models, middleware, utils, services, websocket) and every frontend service is now under test. All routes mounted under `/api` in `server.js` covered.
