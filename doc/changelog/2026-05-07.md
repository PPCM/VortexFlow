# Changelog - 2026-05-07

## Summary
Dead-code cleanup pass on `GraphViewer` (FAB info, sidebar drawer with the
unwired ViewControls, useless `useWebSocket`, `initializeVisualization`
placeholder, dozens of unused MUI imports). ESLint warnings driven to zero
across both backend and frontend. GitHub Actions CI workflow added to run
backend lint + tests and frontend lint + typecheck + tests + build on every
push and pull request. Bundle split: Three.js stack and CodeMirror moved
into their own cache-friendly chunks via `vite.config.ts` `manualChunks`.
The renderer's particle-material patch interval now self-stops after 5
seconds instead of running at 5 Hz forever.

## Changes

- [frontend/src/components/graphs/GraphViewer.tsx] Major cleanup
  (553 → 174 lines, -379). Removed: floating Info FAB, right Drawer with
  the inert `GraphStats` / `ViewControls` panel (its display switches and
  sliders weren't connected to `GraphRenderer3D`), the placeholder
  `initializeVisualization()` console.log, the `useWebSocket` connect /
  disconnect lifecycle (sims are 100% client-side now), the `Settings`
  menu (redundant with the renderer's right Drawer), and the placeholder
  `Reset view` / `Share` / `Export` buttons that only logged. The header
  now keeps only what's wired up: title + description, simulation
  toggle, fullscreen, and (for editors) the Edit shortcut. The
  countDotElements helper is gone with the GraphStats card it fed.
- [frontend/src/components/graphs/GraphRenderer3D.tsx] Removed the dead
  `linkLabelCallback` that only console.log'd, the unused `Position`
  interface, the unused `graph` reference inside the overlay update
  effect, and the unused `EffectsIcon` import along with ~14 other
  unused MUI icons. The runtime particle-material patch interval now
  self-stops after 5 seconds (previously ran at 5 Hz forever); the
  comment explains the rationale.
- [frontend/src/components/graphs/GraphRenderer3D.test.tsx] Removed
  unused `Mock` type import.
- [frontend/src/components/dashboard/Dashboard.tsx] Dropped the now-dead
  `useSimulation` consumer: `simulationActions.loadSessions()` calls a
  backend route that was deleted in the simulation migration, so the
  fallback computation was operating on a permanently empty list.
  Initialization now loads graphs + dashboard stats in parallel and
  defaults `activeSimulations` to 0 in the offline fallback. Removed
  unused `authState` and the unused `graphsResult` / `sessionsResult`
  destructuring slots.
- [frontend/src/components/admin/AdminPanel.tsx] Added
  `// eslint-disable-next-line react-hooks/exhaustive-deps` to the three
  load-on-mount / load-on-tab-change effects: their deps lists are
  intentional (react to a single trigger only), correcting them would
  cause re-fetch loops.
- [frontend/src/components/admin/BulkActionsBar.tsx,
   frontend/src/components/admin/PasswordResetDialog.tsx,
   frontend/src/components/common/ErrorBoundary.tsx,
   frontend/src/components/common/NotificationProvider.tsx,
   frontend/src/components/graphs/DOTCodeMirrorEditor.tsx,
   frontend/src/components/graphs/DOTEditor.tsx,
   frontend/src/components/graphs/GraphEditor.tsx,
   frontend/src/components/graphs/GraphEditor.test.tsx,
   frontend/src/components/graphs/GraphList.tsx,
   frontend/src/components/user/UserProfile.tsx,
   frontend/src/context/GraphContext.tsx,
   frontend/src/context/GraphContext.test.tsx,
   frontend/src/services/errorHandler.ts,
   frontend/src/services/websocket.ts] Removed unused MUI imports,
  unused icon imports, unused type imports, unused destructured fields,
  and a `[\-+]` → `[-+]` regex fix (no-useless-escape).
- [frontend/eslint.config.js] `@typescript-eslint/no-unused-vars` now
  ignores caught errors named `_`, `err`, or `error` — the project's
  convention is to keep the binding for stack traces even when the
  handler doesn't reference it.
- [backend/server.js] Replaced `const io = socketIo(server, {...})`
  with a side-effect call: the WebSocket server still attaches its
  handlers, but the unused `io` reference (left over from the
  SimulationHandler removal) is gone.
- [.github/workflows/ci.yml] (new) GitHub Actions CI: runs on push and
  pull-request to `main`. Backend job: `npm ci` + `npm run lint` +
  `npm test --ci --runInBand`. Frontend job: `npm ci --legacy-peer-deps`
  + `npm run lint` + `tsc --noEmit` + `vitest run` + `vite build`.
  Node 20, npm cache keyed on each package's lockfile.
- [frontend/vite.config.ts] Added a `rollupOptions.output.manualChunks`
  function that pulls Three.js + 3d-force-graph + three-spritetext into
  a `three-*.js` chunk and CodeMirror packages into a `codemirror-*.js`
  chunk. Both are large and only needed inside lazy-loaded routes
  (renderer / editor), so isolating them makes them cache-friendly
  across sessions and avoids duplication when a session visits both
  routes. `chunkSizeWarningLimit` raised to 800 kB to silence the
  no-op warning on the now-isolated `three` chunk.
